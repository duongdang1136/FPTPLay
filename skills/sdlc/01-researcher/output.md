# Researcher Output Contract — FPTPlay

## Default feature-docs output

```text
features/lightweight/<Large-Feature>/<Sub-Feature>/research/<feature>-research.md
```

## Optional strict task-state output

```text
state/tasks/{TASK-ID}/research_report.md
state/tasks/{TASK-ID}/architecture_report.md
```

Use task-state output only when a formal SDLC task runner created a task state.

## Must include

- Relevant knowledge found.
- Product/domain context.
- Architecture/system boundaries when applicable.
- Data/domain model notes when applicable.
- Risks, assumptions, and constraints.
- Suggested downstream focus for BA and Document Writer.
