# Researcher Input Contract — FPTPlay

## Required inputs

- Original user intent and feature/sub-feature name.
- Existing FPTPlay docs under `features/lightweight/**` and `features/final-docs/**`.
- Relevant repository context from this repo.

## Optional inputs

- `state/tasks/{TASK-ID}/state.yaml` when a strict SDLC task state exists.
- Existing Wiki/RAG hits from the active OpenClaw workspace.
- External research when local knowledge is insufficient.

## Priority

1. Existing FPTPlay repo evidence.
2. Existing feature docs and code-backed/dev-owned docs if present.
3. Workspace Wiki knowledge from the active agent workspace.
4. External sources for important gaps only.
5. Stated assumptions when evidence is missing.
